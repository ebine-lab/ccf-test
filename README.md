# CCF Dashboard

離床時間 (X) と安静心拍数 (Y) の相互相関解析ダッシュボード。

整形外科疾患回復病棟の患者データ（96名分）を対象に、
活動量と睡眠時心拍の間のラグ付き因果関係を探索するツールです。

## 機能

- **CSV読み込み**: ドラッグ＆ドロップでデータ投入
- **96名分のスパークライン表示**: X(離床時間)・Y(心拍数)を重畳表示
- **前処理**:
  - 初日・最終日カット（境界日の不完全データ除去）
  - Y=0 線形補間（測定欠損の補間）
  - 体動指数(K列)閾値フィルタ（体動が大きい日のY補間）
  - FFTローパスフィルタ
  - メディアンフィルタ
  - 全操作 Undo / Redo 対応
- **解析**:
  - 集団平均 CCF(τ) ± SE
  - ACF(X), ACF(Y) — 自己相関構造の診断
  - 偏CCF (AR(1)残差) — 自己相関漏れ除去後の純粋なラグ効果
  - 固定ラグ CCF/偏CCF の患者別分布 — 選択バイアスの検証
  - τ*, r* 分布ヒストグラム
- **CSVエクスポート**: ラグ別統計 + 患者別統計

## CSVフォーマット

| 列 | 内容 |
|----|------|
| A  | 患者ID (例: Subject_001) |
| B  | 入棟からの経過日数 (整数) |
| C  | 日中離床時間 X (min, 整数) |
| D〜J | (任意) |
| K  | 体動指数 |
| L  | 安静時心拍数 Y (bpm, float) |

1行目はヘッダ、2行目以降がデータ。96名分が縦にスタック。

## セットアップ

```bash
npm install
```

## 開発サーバー

```bash
npm run dev
```

ブラウザで http://localhost:5173 を開きます。

## ビルド（デプロイ用）

```bash
npm run build
```

`dist/` フォルダに静的ファイルが生成されます。

## デプロイ

### GitHub Pages

```bash
# dist/ の内容を gh-pages ブランチにデプロイ
npx gh-pages -d dist
```

リポジトリの Settings → Pages → Source を `gh-pages` ブランチに設定。

### Netlify / Vercel / Cloudflare Pages

リポジトリを接続し、ビルドコマンドに `npm run build`、
出力ディレクトリに `dist` を指定するだけで自動デプロイされます。

## プライバシー

全計算はブラウザ内で完結します。
患者データがサーバーに送信されることはありません。
